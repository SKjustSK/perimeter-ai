# Perimeter AI – Real‑time Person Re‑Identification Pipeline

## Overview

Perimeter AI is a containerised end‑to‑end system that performs **real‑time person re‑identification (ReID)** on video streams.  It combines modern computer‑vision models (YOLOv8 n for detection and a ResNet‑18 ONNX model for feature extraction) with a scalable backend and a user‑friendly dashboard.

The project is fully **Docker‑compose** ready and consists of four main components:

1. **Inference Service** – FastAPI server that receives video frames, runs object detection & feature extraction, and stores embeddings in Qdrant.
2. **Backend API** – Express/Prisma service exposing REST endpoints for camera management, querying embeddings, and persisting data.
3. **Frontend Dashboard** – React/Vite UI showing live camera maps, searchable person panels, and visualisation of detection results.
4. **Database & Vector Store** – PostgreSQL for metadata and Qdrant for fast similarity search.

The repository ships with scripts to automatically generate realistic Git history spanning the last six months, so the repo looks like a mature project.

---

## Features

- **Real‑time video ReID** using YOLOv8 n + ResNet‑18 ONNX pipeline.
- **Batch feature extraction** offloaded to a `ThreadPoolExecutor` for high throughput.
- **FastAPI inference service** with automatic ONNX model export on container start.
- **Express + Prisma backend** with PostgreSQL persistence and migration support.
- **Qdrant vector store** for efficient similarity search of person embeddings.
- **React dashboard** with live camera map, searchable person panel, and status indicators.
- **Docker‑Compose orchestration** for all services (frontend, backend, inference‑service, db, qdrant, shared volumes).
- **Robust CI‑ready setup** – includes `.gitignore` rules for node modules, Python caches, logs, IDE files, Docker‑mounted data, and auto‑generated model weights.
- **Realistic commit history** generated by `create_commits.py` (last commit dated Dec 10 2025).
- **Easy deployment** – push the repository to GitHub and spin up the stack with a single `docker compose up`.
